# Ideas backlog (parked, not built)

Over-the-top / speculative features kept as ideas only, so the core stays lean.
Promote one to the roadmap only when it earns its cost against a real goal.

## Game / office
- Full tile-map walking (Pokémon-grade pathfinding) — current desk-slot glide is
  enough for ≤12 characters; revisit only past ~15 with a Canvas2D upgrade.
- Custom sprite/image uploads — deferred (sizing/safety/pixel-consistency). Bundled
  sprites only for now.
- Day/night office lighting, weather, minigames, furniture shop.
- Avatar emotes beyond activity states.

## Company layer
- Real-time conversational multi-agent meetings — rejected by design; meetings are
  parallel memos → one synthesis (cheaper + honest). Keep async-only.
- Per-employee always-on processes — rejected; employees are dormant identities on a
  shared runtime (cost = decisions, not headcount).
- A standing multi-agent HR "team" — HR is one policy engine + episodic reviewer.
- Reflection after every task — only on novelty/failure/surprise.
- Company-wide "org chart editor" GUI — later; roles come from the manifest first.

## Deferred integrations
- Cloud/multi-machine company runtime (local-first first).
- Slack/Discord/email delivery for the CEO queue (start with the in-app panel).
- Fine-grained per-tool cost attribution dashboards.

Rule of thumb (from the Codex/Grok deliberation): if a feature adds tokens or
surface area without moving a Key Result, it stays here as an idea.
